Overview

The CMSpace is the entry point, here you create the bodies and setup the space.

Tasks

Other Methods

  • cpSpace The Chipmunk cpSpace.

Initialization

  • - init Standard initialization method.
  • - setGravity: Sets the gravity of the Chipmunk world.
  • - setDamping: Amount of viscous damping to apply to the space. A value of 0.9 means that each body will lose 10% of itís velocity per second. Defaults to 1. Like gravity can be overridden on a per body basis.
  • - setSleepTimeThreshhold: Elapsed time before a group of idle bodies is put to sleep (defaults to infinity which disables sleeping).
  • - setIterations: Allow you to control the accuracy of the solver. Defaults to 10.

  • - updateShapes Updates all SPDisplayObject objects that are attached to the bodies.

Adds a default collision handler which will respond to all collisions.

A selector should have the following signature:

  • (void) collisionListenerBetweenBallAndSquare:(CMArbiter)arbiter space:(CMSpace)space;

The only selector that expects back a BOOL is the begin selector. So all the other selectors can be void.

About the cpCollisionType type, the cpCollisionType type is by default 'id' (keyword), you can override this by changes the CMTypes.h file.

setIterations:

Allow you to control the accuracy of the solver. Defaults to 10.

- (void)setIterations:(int)iterations

Parameters

iterations
the iterations

Discussion

Allow you to control the accuracy of the solver. Defaults to 10.

Chipmunk uses an iterative solver to figure out the forces between objects in the space. What this means is that it builds a big list of all of the collisions, joints, and other constraints between the bodies and makes several passes over the list considering each one individually. The number of passes it makes is the iteration count, and each iteration makes the solution more accurate.

If you use too many iterations, the physics should look nice and solid, but may use up too much CPU time. If you use too few iterations, the simulation may seem mushy or bouncy when the objects should be solid. Setting the number of iterations lets you balance between CPU usage and the accuracy of the physics. Chipmunk’s default of 10 iterations is sufficient for most simple games.
